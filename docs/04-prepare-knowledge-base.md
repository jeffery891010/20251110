# 04｜準備知識庫（教材）

## 目的
- 將「教案與教材」整理為可嵌入的數位檔，並上傳到 repo 的 `data/`。

## 檔案格式建議
- 首選：`.md`、`.txt`（解析最簡單）
- 其他：`.pdf`、`.docx`（已內建解析，上傳後會自動轉為文字處理）

## 命名與切分建議
- 以主題/章節命名檔案，例如：`課程A_單元1.md`
- 內容內部以自然段落或標題層級切分，利於後續分塊（chunking）。

## 上傳方式（無需終端機）
- 推薦：部署完成後，直接到 Web 管理介面 `/admin` 進行上傳（兩種方式）
  - 檔案上傳：支援 `.pdf/.txt/.md/.docx`（伺服器自動轉文字並分塊；支援多檔批次上傳，後端串行嵌入）
  - 貼上純文字：將內容貼到文字框即可
- （選配）CI 自動嵌入：亦可把檔案放入 `data/`，由 CI 監聽變更觸發嵌入（見 `docs/11-ci-cd-with-github-actions.md`）。

## 資料清理
- 移除機密、個資與無意義符號；必要時以掩碼處理。
- 若 .docx 解析失敗，系統會在上傳結果中回報錯誤訊息（例如：檔案損毀、不支援的編碼等）。
